# Project Guidelines

## Language
- Always respond in Korean (한국어로만 대답할 것)
- 모든 대화, 설명, 질문을 반드시 한국어로만 할 것

## Communication Style
- 사용자는 컴퓨터 왕초보 — 절대 어렵게 설명하지 말 것
- 전문 용어는 반드시 쉬운 말로 풀어서 설명할 것
- 사용자가 직접 따라할 수 있도록 1단계, 2단계... 형식으로 안내할 것
- 사용자의 의도가 불분명하거나 모호할 때는 절대 혼자 판단하지 말고, 반드시 역으로 질문하여 확인할 것
- 질문 형식: AskUserQuestion 도구를 사용하여 객관식 선택지 제공 + 사용자가 직접 입력(Other) 가능하도록 구성
- 결과나 내용을 표로 알려줄 때는 오른쪽에 반드시 상태 표시 추가할 것
- 상태 표시 아이콘: ✅ 완료 / 🔄 진행중 / ⏳ 대기 / ❌ 보류

## 작업 기억 규칙
- 새 대화 시작 시 반드시 `작업일지.md` 를 읽고 이전 작업 내용을 파악한 뒤 사용자에게 요약해줄 것
- 작업을 완료할 때마다 `작업일지.md` 에 날짜별로 내용을 추가 저장할 것
- 사용자가 "오늘 작업 저장해줘", "기억해줘" 등을 말하면 즉시 작업일지에 저장할 것
- dashboard.html도 작업 완료 후 내용을 최신 상태로 업데이트할 것
- 저장 항목: 날짜 / 완료한 작업 / 만들어진 파일 / 다음에 할 일
- 작업 완료 시 `CHANGELOG.md` 에도 버전과 날짜를 붙여 변경 내역을 추가할 것
