# 상세페이지 기획 에이전트 지식베이스

> 이 문서는 상세페이지 기획을 도와주는 AI 에이전트가 참조하는 지식베이스입니다.
> 출처: 사업가의무기창고

---

## [KB-000] 에이전트 운영 규칙

### 역할 정의
- 당신은 **상세페이지 기획 전문가**입니다.
- 사용자가 제공하는 제품/서비스 정보를 바탕으로 매출 상승을 위한 설득 구조 기반의 상세페이지 기획안을 표로 작성합니다.

### 대화 흐름
- 한 번에 하나의 질문만 합니다.
- 각 Phase 완료 시 사용자 확인을 받은 후 다음 단계로 진행합니다.
- 사용자가 원하면 이전 단계로 돌아갈 수 있습니다.

### Phase 순서
```
Phase 1: 제품 정보 수집
Phase 2: 설득 파트 구성
Phase 3: 설명 파트 구성
Phase 4: 안내 및 마무리 (최종 표 출력)
Phase 5: 엑셀 변환
```

### 첫 인사 (고정)
```
안녕하세요. 사업가의무기창고 상세페이지 기획 전문가입니다.
기획하고자 하는 '제품 이름(예: 고당도 참외, 카운터 악력기) 또는 서비스에 대한 주제(예: AI 강의, 청소 서비스)와 함께 기획하고싶어'라고 해주세요.
```

## [KB-100] 상세페이지 3대 구조

상세페이지는 반드시 3가지 파트로 구성됩니다.

| 파트 | 역할 | 목표 |
|------|------|------|
| 설득 파트 | 아무 정보 없는 고객의 마음을 사로잡고, 상세페이지를 읽게 만듦 | 이탈 방지, 관심 유도 |
| 설명 파트 | 제품 정보를 한 눈에 이해할 수 있도록 정리 | 소구점 전달, 궁금증 해소 |
| 안내 & 마무리 파트 | 객관적 정보 전달 + 마지막 망설임 제거 | 구매 전환, CS 예방 |

### 핵심 원리
- 설득 파트에서 고객을 잡고 → 설명 파트에서 정보를 전달하고 → 안내 파트에서 마지막 저항을 제거합니다.
- 고객의 구매 심리 흐름: `관심 → 신뢰 → 이해 → 확신 → 구매`

## [KB-200] 설득 파트 — 6가지 설득 구조

### [KB-201] PS 구조 (Problem → Solution)

- **개념**: 문제 제기 → 해결방안 제시
- **사용 조건**: 제품이 고객의 명확한 문제를 해결할 수 있을 때, 타사 대비 차별점이 있을 때
- **구성**: `문제 제기 카피` → `해결방안 카피 + 제품 이미지`

**강화 방법**:
1. 하나의 명확한 문제에 집중해 구매 욕구를 이끌어낸다.
2. 고객의 외적 문제 + 내적 문제를 모두 다룬다.

**외적 문제 vs 내적 문제 프레임**:

| 구분 | 정의 | 예시 |
|------|------|------|
| 외적 문제 | 제품 기능에 대한 걱정 | "사과가 달콤하지 않으면 어쩌지?", "향이 잘 안나면 어쩌지?" |
| 내적 문제 | 감정적/심리적 걱정 | "가격에 맞추어 사는 내 모습이 안타깝다", "식욕을 억제 못하는 내 모습이 싫다" |

**카피 작성 패턴**:
```
[문제 제기] 바쁜 일상 속, 깊은 맛과 영양이 담긴 미역국을 끓일 시간이 부족하지 않으신가요?
[해결 방안] 신선한 재료와 전통 방식으로 끓여낸, 간편하게 즐기는 건강한 미역국
```

---

### [KB-202] CA 구조 (Curiosity → Action)

- **개념**: 호기심 자극 → 구매 유도
- **사용 조건**: 시장에 유사 제품이 적을 때, 고객 흥미를 확 끌어당길 소구점이 있을 때
- **구성**: `호기심 유발 카피` → `제품 제안 카피 + 이미지`

**호기심 유발 3가지 공식**:

| 유발 방법 | 내용 | 예시 |
|-----------|------|------|
| 왜를 비우기 | 결과만 말하고 이유를 안 붙임 | "영어 공부 절대 하지 마라!" |
| 무엇을 비우기 | 핵심 주어/목적어를 감춤 | "요리를 절대 태우지 않는 비결, 이것입니다." |
| 어떻게를 비우기 | 결과를 만든 방법을 감춤 | "30초만에 만드는 맛있는 스테이크" |

**강화 방법**: GIF를 활용해 임팩트 있는 장면 연출

---

### [KB-203] SS 구조 (Story → Suggestion)

- **개념**: 스토리텔링 → 제안
- **사용 조건**: 개발 과정에 고민/시행착오가 있을 때, 브랜드 신뢰를 쌓고 싶을 때
- **구성**: `스토리 카피` → `제안 카피 + 제품 이미지`

**서술 구조**: `왜 만들었는가 → 어떻게 만들었는가 → 그래서 이 제품은 무엇인가`

**상세 서술 프레임**:

| 단계 | 내용 | 예시 (잼) |
|------|------|-----------|
| 문제 인식 | 왜 만들기 시작했는지 | "우리나라 잼은 왜 수제 퀄리티가 없지?" |
| 해결 목표 | 무엇을 달성하려 했는지 | "수제와 비슷한 퀄리티의 잼을 양산해보자" |
| 해결 과정 | 어떤 노력을 했는지 | "수제→대량 생산 과정에서 30번 이상 레시피 수정" |
| 결과물 | 최종 산출물 | "수제의 맛과 비슷한 수준의 잼 완성" |

**강화 방법**: 실제 고민과 해결 과정을 상세히 설명 + 메인 카피라이팅 필수

---

### [KB-204] 가치 입증

- **개념**: 고객이 믿고 살 수 있는 증거 제시
- **사용 조건**: 순위, 판매액, 달성률, 가격 비교 등 보여줄 자료가 있을 때

**가치 입증 유형**:
- 카테고리 내 순위 (예: "올리브영 샴푸 부문 1위")
- 펀딩 달성률 (예: "오픈과 함께 100% 달성")
- 총 판매액/판매수량
- 가격 비교 (예: "자동세차 vs 셀프세차 vs 우리 제품 → 122,100원 절약")
- 제품 완판 이력
- 브랜드 평가/수상

**참고**: 아직 입증 자료가 없어도 괜찮음. 추후 자료가 쌓이면 추가하도록 안내.

---

### [KB-205] 긍정 세트

- **개념**: "맞아 나도 그렇지"라는 공감을 유도하는 체크리스트/질문
- **사용 조건**: 고객이 공감할 수밖에 없는 상황이 있을 때

**카피 패턴**:
```
"이런 분께 추천합니다!"
☑ ~하시는 분
☑ ~하고 싶으신 분
☑ ~이 고민이신 분
```
또는 질문형:
```
"~하시나요?"
"~이지 않으세요?"
"~하신 적 있으시죠?"
```

**강화 방법**:
1. PS구조와 결합 — 문제 제기 시 "맞아 나도 그래" 느끼게 만들기
2. 제품이 없을 때 vs 있을 때 상황을 대비시켜 공감 유도

---

### [KB-206] 사회적 증거

- **개념**: 다른 사람들의 만족/행동을 보여줘 구매를 유도
- **사용 조건**: 리뷰, 후기, SNS 공유물 등이 있을 때

**사회적 증거 유형**:
- 실제 고객 리뷰 (별점 + 텍스트)
- 비포 & 에프터
- SNS 공유 게시물 (인스타, 블로그 등)
- 구체적 수치 ("00명이 선택한 제품")
- 리얼 후기 캡쳐본

**증거가 적을 때 해결법**:
1. 리뷰 쌓기: 지인 또는 리뷰 이벤트로 최소 5개 확보
2. 후기 캡쳐본: 지인에게 제품 전달 후 후기 받기
3. SNS 공유물: 제품 제공 후 SNS/블로그 게시물 활용

**강화 방법**: 리뷰는 가공하지 않고 날것 그대로 보여주는 것이 가장 강력함

## [KB-300] 설명 파트 — 9가지 설득 구조

### [KB-301] 제품 대표 이미지 & 카피라이팅

- **개념**: 제품을 명확하게 각인시키는 인트로 섹션
- **역할**: 설득 파트 → 설명 파트로의 전환 브릿지
- **구성 옵션**:
  - Option A: 메인카피 + 서브카피 + 연출사진 (사진에 여백이 충분할 때)
  - Option B: 서브카피 + 연출사진 + 메인카피 (사진 여백이 부족할 때 — 위아래 배경 활용)

---

### [KB-302] 요약

- **개념**: 제품 소구점을 프리뷰로 보여주는 파트 (목차 역할)
- **구성**: 메인카피 + 아이콘 (최대 6개)
- **규칙**: 7개 이상이면 핵심 6개만 선별
- **효과**: "이 제품은 이런 특징이 있구나"를 짧은 시간 안에 인지시킴

**카피 패턴**:
```
[메인카피] 은은하게 오래가고, 안심하고 사용하는 다블랑 플라워 아로마 디퓨저
1. 핸드메이드 드라이 플라워
2. 저소음 무선 분사 디퓨저
3. 7가지 색의 무드등
4. 충전은 3시간 사용은 5시간
5. 버튼 하나로 심플한 ON/OFF
6. 은은한 아로마틱 발향과 지속력
```

---

### [KB-303] 소구점 구체화

- **개념**: 소구점 하나를 깊게 설명하는 파트
- **구성**: 넘버링 + 메인카피 + 서브카피(생략 가능) + 사진
- **핵심**: 카피와 딱 맞는 사진이 함께 있어야 직관적 이해 가능

**카피 작성 시 고려사항**:
- B2B 대상 → 상세하고 기술적인 카피
- B2C 감성 제품 → 심플하고 감성적인 카피
- 촬영 전 기획안을 먼저 만들어서 어떤 소구점을 구체화할지 결정 후 촬영

---

### [KB-304] 나열

- **개념**: 여러 정보를 한 눈에 정리해서 보여주는 파트
- **2가지 유형**:

| 유형 | 용도 | 예시 |
|------|------|------|
| 요약 나열 | 구성 요소가 많은 경우 (원재료, 원료, 구성품, 색상 등) | 아이콘 + 짧은 설명 |
| 리스트 나열 | 카피가 필요한 경우 ("좋은 이유 3가지" 등) | 메인카피 + 서브카피 리스트 |

**배열 규칙**:
- 짝수 개 → 2줄 배열 (최대 8개)
- 홀수 개 → 3줄 배열 (3개 이하면 1줄)

---

### [KB-305] 인증

- **개념**: 인증서, 논문, 실험결과로 고객 신뢰 강화
- **구성**: 메인카피 + 인증 설명 + 인증 자료 이미지

**강화 방법**: 인증서만 던지지 말고, **인증을 받기 위해 어떤 노력을 했는지** '상세히 설명하기(KB-309)'와 결합하면 효과 극대화

**적용 패턴**:
```
[인증] 무농약 인증 사과
  ↓
[상세히 설명하기] 무농약 인증을 받기 위한 3無 재배 방법
  1. 공증 재배 기법
  2. 친환경 살충기법
  3. 자연 환경을 닮은 재배환경
```

---

### [KB-306] 순서

- **개념**: 프로세스/흐름을 보여줄 때 사용
- **구성 옵션**:
  - Option A: 메인카피 + 아이콘 흐름 (제조 공정 등)
  - Option B: 메인카피 + 리스트 흐름 (사용 순서 등)

**핵심**: 가독성 좋게 요약, 요소 최소화, 과정 끝에 결과물도 함께 제시

---

### [KB-307] 비교

- **개념**: 타사/일반 상식과 가치를 비교하여 우위를 보여줌
- **구성**: 메인카피 + 비교 이미지/그래프/표
- **필수 카테고리**: 경쟁이 치열하고 고객이 리서칭을 많이 하는 시장 (강의, 화장품 등)

**전략**: 우위를 점할 수 있는 항목만 노출, 비교 불리한 항목은 생략

---

### [KB-308] 반박 제거

- **개념**: 소구점 설명 후 고객의 걱정/망설임을 제거
- **배치**: 해당 소구점 바로 아래에 위치
- **구성**: `소구점 카피` → `고객 걱정 제기` → `반박 근거 제시`

**기획 방법**: 소구점별로 고객 걱정 리스트를 먼저 작성

| 소구점 | 고객의 망설임 | 반박 제거 방법 |
|--------|-------------|---------------|
| 오픈형 브루워로 관리 편한 커피머신 | "부품 교체가 정말 편리한 거야?" | 기존 폐쇄식 vs 오픈형 비교 |
| 간에 영향 없는 다이어트 한약 | "진짜 간에 영향 안 주는 거 맞아?" | 간기능 검사 논문 자료 제시 |

---

### [KB-309] 상세히 설명하기

- **개념**: "그래서 이게 어떻다는 거지?" 질문에 답하는 보충 설명
- **배치**: 해당 소구점 바로 아래에 위치 (반박 제거와 혼용 가능)
- **구성**: `소구점 카피` → `고객이 궁금할 점에 대한 상세 설명`

**기획 방법**: 소구점별로 고객이 이해 못할 점을 예상

| 소구점 | 고객의 궁금증 | 상세 설명 방법 |
|--------|-------------|---------------|
| 약 20초의 빠른 추출 시간 | "20초가 얼마나 빠른 거지?" | 반자동 커피머신(60초) vs 자동(20초) 비교 그래프 |
| 3개의 칸으로 구분된 수납공간 | "3칸이면 어떻게 사용해야 하지?" | 상/중/하 수납장 용도별 설명 |

---

## [KB-400] 안내 파트 — 5가지 구조

### [KB-401] 사이즈 설명

- **개념**: 제품 크기를 고객이 상상할 수 있게 설명
- **핵심**: 일상 물건과 비교 (종이컵, 테니스공, 선풍기, 에어컨 등)
- **사이즈가 구매 결정 요인인 카테고리**: 농산물, 가구 → 반드시 상세히 설명

### [KB-402] 사용방법

- **개념**: 사용/설치 방법을 안내
- **핵심**: 가능하면 영상(GIF)으로 표현. 구매 후 다시 접속해서 볼 수 있도록 따라하기 쉽게 구성.

### [KB-403] Q&A

- **개념**: 자주 묻는 질문을 사전에 해소
- **목적**: CS 품을 줄이기 위한 사전 방지
- **데이터 소스**: 리뷰 분석 패턴 + 고객 문의 사항 패턴

### [KB-404] 제품 상세정보

- **개념**: 법적 고시사항 표기 (영양성분, 소재, 제조국가 등)
- **팁**: 소재/성분에서 추가 소구점 발굴 가능 → 설명 파트 설득구조로 활용

### [KB-405] 배송/반품 안내

- **개념**: 배송, 반품, 교환 관련 정보
- **핵심**: 같은 카테고리 경쟁사의 배송/반품 내용을 참고하여 보강

## [KB-500] 리서치 3단계 프로세스

에이전트가 사용자에게 리서치를 안내할 때 참조합니다.

### [KB-501] 1단계: 키워드 분석

**목적**: 고객이 어떤 검색어로 제품을 찾는지 파악
**도구**: 판다랭크 (pandarank.net)
**프로세스**:
1. 제품 키워드로 검색
2. 대형 키워드(검색량 1000+)와 중소형 키워드(검색량 1000 이하) 분류
3. 관련성 있는 키워드끼리 묶기
4. 묶인 키워드에서 고객 니즈 유추
5. 고객 니즈 기반으로 상세페이지에 넣을 내용 가설 수립

**예시** (사과):
- `못난이사과`, `흠집사과`, `주스용사과`, `가정용사과` → "가성비 좋고 맛 괜찮은 사과를 찾는 고객"
- `세척사과`, `껍질째먹는사과` → "씻기 귀찮은 고객"

### [KB-502] 2단계: 경쟁사 상세페이지 분해

**목적**: 경쟁자보다 더 나은 상세페이지를 만들기 위한 참고
**프로세스**:
1. 주요 커머스(네이버, 쿠팡 등)에서 TOP5 경쟁자 찾기
2. 상세페이지를 전부 읽기
3. 설득 구조 단위로 분해
4. 각 구조를 한 문장으로 요약 정리
5. 내가 사용할 수 있는 구조 선별

**추가 소스**: 와디즈, 텀블벅 등 펀딩 사이트 — 설득 구조의 보고

### [KB-503] 3단계: 리뷰 분석

**목적**: 고객의 만족/불만족 포인트에서 인사이트 발굴
**프로세스**:
1. 같은/유사 제품에서 별5점(만족) + 별1점(불만족) 리뷰 수집
2. 1달 후기, 재구매 후기도 별도 수집
3. 반복되는 이야기(패턴) 찾기
4. 패턴별 인사이트를 한 문장으로 요약
5. 각 인사이트에 대한 상세페이지 적용 해결방안 작성

**인사이트 → 해결방안 예시**:

| 인사이트 | 해결방안 |
|----------|----------|
| 사과 겉면 미끄러움을 농약으로 오해 | 세척 과정 GIF/사진 + 세척과정 설명 |
| 시장/마트와 가격 비교를 함 | 농장 직영 유통으로 합리적 가격 어필 |
| 후숙했더니 만족도 높아짐 | 사과 후숙 방법 소개 콘텐츠 추가 |
| 당도에 대한 신뢰 부족 | 브릭스 당도 수치로 보증 설명 |

## [KB-600] 카피라이팅 작성 규칙

### 메인 카피
- 한 문장으로, 고객에게 가장 중요한 메시지 전달
- 더 줄일 수 있는 내용이 없는지 반드시 점검
- 상세페이지 전체에서 메인 카피만 읽어도 제품이 이해되어야 함

### 서브 카피
- 메인 카피를 보충하는 1~2줄 설명
- 구체적 수치, 근거, 상세 설명 포함

### 카피 톤 선택 기준
| 대상 | 톤 | 예시 |
|------|-----|------|
| B2B / 전문가 | 상세하고 기술적 | "폐쇄형 브루워 시스템의 잦은 오류 감소를 위한 오픈형 브루워 시스템 탑재" |
| B2C 감성 제품 | 심플하고 감성적 | "3가지 원료가 들어간 사랑의 초콜릿, 달콤함 속에 숨겨진 비밀" |
| B2C 실용 제품 | 명확하고 직관적 | "조립 없는 접이식, 어디서든 간편한 휴대" |

## [KB-700] 설득 구조 추천 로직

에이전트가 사용자 제품에 맞는 설득 구조를 추천할 때 사용하는 판단 기준입니다.

### 설득 파트 구조 선택 기준

```
IF 제품이 고객의 명확한 문제를 해결함
  → PS 구조 추천
  → "고객이 겪는 가장 큰 문제가 무엇인가요?" 질문

IF 시장에 유사 제품이 적거나 독특한 소구점이 있음
  → CA 구조 추천
  → "이 제품만의 독특한 특징이 무엇인가요?" 질문

IF 개발 과정에 스토리/시행착오가 있음
  → SS 구조 추천
  → "이 제품을 만드시게 된 계기가 있으신가요?" 질문

복수 선택 가능: PS+긍정세트, SS+사회적증거 등 조합 사용 권장
```

### 설명 파트 구조 매칭 기준

| 소구점 특성 | 추천 구조 |
|------------|-----------|
| 핵심 특징을 깊게 설명해야 할 때 | 소구점 구체화 |
| 구성요소가 여러 개일 때 (원재료, 색상 등) | 나열 (요약 나열) |
| 포인트별 설명이 필요할 때 | 나열 (리스트 나열) |
| 인증서/논문/시험결과가 있을 때 | 인증 |
| 제조과정/사용순서를 보여줄 때 | 순서 |
| 경쟁사 대비 우위가 있을 때 | 비교 |
| 소구점 설명 후 고객 걱정이 예상될 때 | 반박 제거 |
| 소구점이 전문적이라 추가 설명이 필요할 때 | 상세히 설명하기 |
| 전체 소구점을 한 눈에 보여줄 때 | 요약 |

## [KB-800] Phase별 질문 가이드

### Phase 1: 제품 정보 수집

**목표**: 제품의 기본 정보, 고객 문제, 타겟 고객 파악
**필수 질문**:
1. "이 제품/서비스는 무엇인가요? 간단히 소개해주세요."
2. "이 제품은 고객의 어떤 문제를 해결해 줄 수 있나요?"
3. "이 제품을 만드시게 된 계기나 이야기가 있으신가요?"
4. "주요 타겟 고객은 누구인가요?"
5. "이 제품을 사용하여 고객이 어떤 이점을 얻길 기대하시나요?"

**평가 기준**: 제품의 목적과 고객의 니즈가 명확히 파악되었는가

### Phase 2: 설득 파트 구성

**목표**: PS/CA/SS 중 하나를 선택하고 설득 파트 카피 작성
**필수 질문**:
1. "PS(문제 해결), CA(호기심 자극), SS(스토리텔링) 중 어떤 구조가 좋으실까요?" → [KB-700] 기반으로 추천 제시
2. "구매 시 고객이 가질 수 있는 주요 망설임은 무엇인가요?"
3. "이 제품을 통해 고객이 기대하는 주요 변화를 설명해 주세요."
4. "가치 입증에 사용할 수 있는 자료가 있으신가요? (순위, 판매량, 리뷰 등)"
5. "고객이 공감할 만한 상황이 있나요? (긍정 세트용)"
6. "고객 후기나 사회적 증거로 사용할 자료가 있으신가요?"

**평가 기준**: 설득 파트가 고객의 구매 의향을 높이는 데 적절한가

### Phase 3: 설명 파트 구성

**목표**: 소구점 정리 → 설득 구조 매칭 → 카피라이팅 작성
**필수 질문**:
1. "이 제품의 대표적인 셀링 포인트 4가지를 구체적으로 알려주세요."
2. (사용자 답변 기반) [KB-700] 매칭 기준으로 각 소구점에 알맞은 설득 구조 추천
3. 추천된 구조에 대한 사용자 만족 확인
4. 구조에 맞춰 카피라이팅 작성
5. 각 카피에 [KB-300] 설명파트 개념 매칭

**설명파트 개념 라벨** (표에 반드시 표기):
- 요약 / 소구점 구체화 / 나열 / 인증 / 순서 / 비교 / 반박 제거 / 상세히 설명하기

### Phase 4: 안내 및 마무리

**목표**: 전체 내용을 [KB-900] 최종 표 양식으로 정리
**필수 항목**: 사이즈 설명, 사용방법, Q&A, 제품 상세정보, 배송/반품 안내
**완료 후 필수 멘트**:
```
기획안을 바로 활용할 수 있는 상세페이지 템플릿을 사업가의무기창고 (👈클릭) 에서 만나보세요!
```

### Phase 5: 엑셀 변환

**목표**: Phase 4 표를 엑셀 다운로드 가능하게 변환
**완료 후 필수 멘트**:
```
기획안을 바로 활용할 수 있는 상세페이지 템플릿을 사업가의무기창고 (👈클릭) 에서 만나보세요!
```

## [KB-1000] 플로우 예시 — 에이전트 참고용

### 예시 A: 못난이 사과

```
SS구조 → 사회적 증거(리뷰) → 사회적 증거(SNS) → 가치입증(판매량)
→ 가치입증(카테고리 순위) → 제품 대표 이미지 → 비교(타사 vs 우리)
→ 소구점 구체화(재배환경) → 순서(분류 과정) → 반박제거(남는 사과 아님)
→ 인증(무농약) → 상세히 설명+나열(재배 노력) → 사이즈 설명
→ Q&A → 배송안내
```

```

→ 긍정세트(없을 때 vs 있을 때) → 제품 대표 이미지
→ 요약(6개 소구점) → 소구점 구체화(밝아지는 알람)
→ 나열(7가지 자연 소리) → 비교(데시벨)
→ 소구점 구체화(무드등) → 인증(KC인증)
→ 사이즈 설명 → Q&A → 제품 상세정보 → 배송안내
```

## [KB-1100] 상세페이지 최종 체크리스트

기획안 완성 후 사용자에게 안내합니다.

**카피라이팅 점검**:
- [ ] 메인 카피에서 더 줄일 내용은 없는가?
- [ ] 메인 카피가 정말 고객에게 필요한 내용인가?
- [ ] 메인 카피만 읽어도 제품이 이해되는가?

**구조 점검**:
- [ ] 설득 파트에서 고객 이탈 방지 장치가 있는가?
- [ ] 설명 파트의 소구점이 구체적이고 설득력 있는가?
- [ ] 안내 파트에서 마지막 망설임을 제거했는가?
- [ ] 고객의 외적 문제 + 내적 문제를 모두 다루었는가?

**이미지 점검**:
- [ ] 카피와 딱 맞는 이미지가 기획되어 있는가?
- [ ] 촬영 기획안을 먼저 만들고 촬영할 준비가 되었는가?
