# 상품등록 서포터 실행

## 시작

이 스킬이 호출되면 다음 순서로 진행합니다.

---

## Step 1: 입력 모드 자동 판단

사용자의 메시지를 분석하여 모드를 결정합니다.

### 엑셀 입력 모드
**트리거**: 파일 경로(.xlsx, .xls, .csv), "엑셀", "파일", "대량" 언급

처리 절차:
1. 파일을 Bash에서 python3 + openpyxl로 읽는다
2. 각 행에서 상품명, 원가, 옵션 등을 추출한다
3. 상호명이 없으면 한 번만 질문한다
4. 각 상품별로 Step 3의 결과를 생성한다

### 도매사이트 모드
**트리거**: "도매", "도매사이트", "사이트에서 찾아", "adminplus" 언급

처리 절차:
1. mcp__claude-in-chrome__tabs_context_mcp로 현재 탭 확인
2. mcp__claude-in-chrome__tabs_create_mcp로 새 탭 생성
3. mcp__claude-in-chrome__navigate로 https://flinc2333.adminplus.co.kr/partner/login.html 접속
4. 로그인 상태 확인 → 미로그인이면 사용자에게 안내:
   "도매사이트 로그인이 필요합니다. 브라우저에서 직접 로그인 후 알려주세요."
5. 로그인 확인 후 사용자가 원하는 상품 검색
6. 상품 정보(상품명, 원가, 옵션, 용량) 추출
7. 추출 정보로 Step 3 진행

### 직접 입력 모드
**트리거**: 위에 해당하지 않는 모든 경우

처리 절차:
1. 사용자 메시지에서 이미 제공된 정보를 파악한다
2. 4가지 필수 정보(상호명, 상품, 원가, 용량/옵션) 중 부족한 것만 질문한다
3. 한 번에 하나씩 질문한다
4. 모든 정보가 모이면 Step 3 진행

---

## Step 2: 선택 정보 확인

사용자가 메시지에서 다음을 언급했다면 반영합니다. 언급하지 않았다면 기본값을 사용합니다.

| 항목 | 기본값 | 사용자 언급 키워드 |
|------|--------|-------------------|
| 마진 모드 | 기본 (x1.8) | "공격적" → x1.5~1.6 / "널널" "여유" → x2.2 |
| 옵션 스타일 | 황금코볼트식 | "베이직" "일반" → 베이직 |
| 상세페이지 | 보통 | "간이" "빠르게" → 간이 / "집중" "풀" → 집중 |
| 플랫폼 | 둘 다 | "올웨이즈만" / "토스만" |

## Step 3: 결과 생성

**반드시 지식베이스를 먼저 읽으세요:**
```
Read knowledge/황금코볼트식_상품등록법.md
Read knowledge/올웨이즈_상품명_태그.md
Read knowledge/상세페이지_기획.md
```

### 3-1. 올웨이즈 상품명 (3개 안)

knowledge/올웨이즈_상품명_태그.md의 규칙을 따릅니다:
- `[★홍보멘트★] + [상호명] + [수식어] + [핵심키워드들(단독+결합)] + [용량/갯수]`
- 35~55자 이내
- 핵심 키워드를 단독 + 수식어 결합 두 형태로 반복
- 각 안의 글자 수를 표시

### 3-2. 토스쇼핑 상품명 (3개 안)

knowledge/황금코볼트식_상품등록법.md의 토스 규칙을 따릅니다:
- 별표 없음
- 키워드 중복 없음
- 깔끔하고 정돈된 설명형

### 3-3. 가격 설정

마진 공식 (식품 기준):
- 공격적: 원가 x 1.5~1.6
- 기본: 원가 x 1.8
- 널널: 원가 x 2.2

예쁜 가격: 뒷자리를 반드시 800원 또는 900원으로 맞춤

플랫폼별 세팅:
- 올웨이즈: 팀구매가 = 예쁜 가격 / 개인구매가 = 팀구매가 x 1.1 + 1원 이상 (예쁜 가격으로 조정)
- 토스: 판매가 = 예쁜 가격 / 정상가 = 판매가보다 높게 (예쁜 가격으로 조정)

### 3-4. 옵션명

**황금코볼트식 (기본)**: 수식어 1~2개를 옵션명 앞에 붙임
- 예: `산지직송 제철 사과 5kg`

**베이직**: 용량+옵션만
- 예: `사과 5kg`

카테고리별 추천 수식어는 knowledge/황금코볼트식_상품등록법.md 참조

### 3-5. 태그 (올웨이즈)

knowledge/올웨이즈_상품명_태그.md의 규칙을 따릅니다:
- 5~6개 생성
- 핵심 키워드 단독 + 수식어 결합
- 쉼표로 구분

### 3-6. 상세페이지 기획표

knowledge/상세페이지_기획.md를 참조하여 선택한 모드에 맞게 생성합니다:

**간이 (4개 섹션)**:
| 섹션 | 카피라이팅 | 이미지 설명 |
|------|-----------|------------|
| 메인 카피 | (한 줄 핵심 메시지) | (제품 연출 사진) |
| 소구점 요약 | (3~4개 아이콘) | (아이콘 이미지) |
| 제품 상세정보 | (법정 고시사항) | (표) |
| 배송/반품 | (안내) | (텍스트) |

**보통 (8~10개 섹션)**:
설득 파트(PS/CA/SS 중 1개 + 긍정세트 또는 사회적증거) + 설명 파트(대표이미지, 요약, 소구점구체화 2~3개, 나열/비교) + 안내 파트(사이즈, Q&A, 상세정보, 배송/반품)

**집중 (15~20개 섹션)**:
설득 파트(PS/CA/SS + 가치입증 + 긍정세트 + 사회적증거) + 설명 파트(대표이미지, 요약, 소구점구체화 3~4개, 나열, 비교, 인증, 순서, 반박제거, 상세히설명하기) + 안내 파트(사이즈, 사용방법, Q&A, 상세정보, 배송/반품)

## Step 4: 출력

다음 형식으로 출력합니다:

```
━━━ 📦 상품 등록 결과 ━━━

📌 올웨이즈 상품명 (3개 안)
  1. (상품명) — OO자
  2. (상품명) — OO자
  3. (상품명) — OO자

📌 토스쇼핑 상품명 (3개 안)
  1. (상품명)
  2. (상품명)
  3. (상품명)

📌 가격 설정
  ┌─────────┬──────────┬──────────┐
  │  모드    │  가격     │  배수    │
  ├─────────┼──────────┼──────────┤
  │ 공격적   │ OO,O00원 │ x1.5~1.6│
  │ 기본     │ OO,O00원 │ x1.8    │
  │ 널널     │ OO,O00원 │ x2.2    │
  └─────────┴──────────┴──────────┘

  올웨이즈: 팀구매가 OO,O00원 / 개인구매가 OO,O00원
  토스쇼핑: 판매가 OO,O00원 / 정상가 OO,O00원

📌 옵션명 (황금코볼트식)
  (옵션1) / (옵션2) / (옵션3)

📌 태그 (올웨이즈)
  #태그1, #태그2, #태그3, #태그4, #태그5

📌 상세페이지 기획 (모드)
  [기획표]

⚠️ 유기농, 무농약, 고당도 같은 단어 및 특정 지역명은 해당 인증을 받은 상품에 대해서만 사용할 수 있으므로, 이로 인한 손해나 리스크는 저희가 책임지지 않습니다. 실제 상품 특성에 맞는 정확한 표현을 사용해주세요.
```

결과 출력 후:
- "수정하고 싶은 부분이 있으면 말씀해주세요."
- "마진 모드, 옵션 스타일, 상세페이지 모드를 변경할 수도 있습니다."
- 사용자가 수정을 요청하면 해당 부분만 재생성합니다.
